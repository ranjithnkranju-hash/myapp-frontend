GitHub Actions Deployment Pipeline Create this file: ```bash .github/workflows/deploy.yml ``` ## deploy.yml ```yaml name: Deploy Static Website to S3 on: push: branches: - main jobs: deploy: runs-on: ubuntu-latest steps: - name: Checkout code uses: actions/checkout@v4 - name: Configure AWS Credentials uses: aws-actions/configure-aws-credentials@v4 with: aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }} aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }} aws-region: ap-south-1 - name: Upload files to S3 run: | aws s3 sync . s3://YOUR_BUCKET_NAME --delete - name: CloudFront Cache Invalidation run: | aws cloudfront create-invalidation \ --distribution-id YOUR_DISTRIBUTION_ID \ --paths "/*" ``` --- # 5. AWS Setup Steps ## Step 1 — Create S3 Bucket Example: ```bash quickbite-demo-app ``` Enable: * Static Website Hosting * Public Read Access (for demo only) --- ## Step 2 — Upload Website Files Upload: * index.html * style.css * app.js --- ## Step 3 — Create CloudFront Distribution Origin: ```bash S3 Static Website Endpoint ``` Enable: * Redirect HTTP to HTTPS * Default Root Object = index.html --- ## Step 4 — GitHub Secrets Add these secrets in GitHub: ```bash AWS_ACCESS_KEY_ID AWS_SECRET_ACCESS_KEY ``` --- # 6. AWS CLI Commands ## Upload manually ```bash aws s3 sync . s3://YOUR_BUCKET_NAME ``` ## CloudFront Invalidation ```bash aws cloudfront create-invalidation \ --distribution-id YOUR_DISTRIBUTION_ID \ --paths "/*" ``` --- # 7.
